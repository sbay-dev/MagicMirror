# Magic Mirror v1.0.3

This release fixes the root cause behind the Sarmad dictionary gateway failures.

## Root cause

The canonical documentation gateway returned:

```text
5028: This model was deprecated on 2026-05-30. Please use an alternative model.
```

It was still deployed against `@cf/openai/gpt-oss-120b` and ignored the `model`
value sent by the native client.

## Fixes

- Updated the Magic Mirror web layer to `@cf/openai/gpt-oss-20b`.
- Migrated native settings from deprecated `@cf/openai/gpt-oss-120b` to
  `@cf/openai/gpt-oss-20b`.
- Removed the broken `wmr-doc.pages.dev` endpoint as a default product fallback.
- When no gateway is configured, translation still uses MT fallback, while the
  dictionary panel clearly reports that a Sarmad gateway must be configured.

## Download

Use `MagicMirror-v1.0.3-windows-x64.zip` from the release assets, extract it,
and run `MagicMirror.Native.exe`.
